# Proyecto Taller Grupal Semana 4

Bienvenido al repositorio central de este taller grupal, donde consolidamos tanto una demostración técnica de integración empresarial como los esfuerzos de Ingeniería y Quality Assurance (QA) enriquecidos por Inteligencia Artificial.

## 📄 Archivos Principales en la Raíz

En la raíz del proyecto encontrarás los siguientes tres (3) documentos fundamentales donde se apoya el proceso de estructuración del ciclo de vida del desarrollo:

- **[`BUSINESS_CONTEXT.md`](./BUSINESS_CONTEXT.md)**
  La plantilla de contexto de negocio completa y diligenciada. Este archivo funcionó como entrada base (prompts y contexto) para alimentar las Gemas u otras herramientas de Inteligencia Artificial que apoyaron la creación del proyecto.

- **[`USER-STORIES-REFINEMENT.md`](./USER-STORIES-REFINEMENT.md)**
  El informe comparativo que reúne las Historias de Usuario (HU). Incluye sus tres facetas principales: la versión original inicial, la versión refinada a través de la Gema (IA) y un cuadro detallado con el registro de las diferencias detectadas y análisis.

- **[`TEST_CASES_AI.md`](./TEST_CASES_AI.md)**
  La matriz global con los casos de prueba generados por las herramientas de IA. Cuenta además con las tablas de los ajustes manuales efectuados por el equipo, plasmando la revisión crítica del QA junto a su debida justificación técnica para su adaptación en el proyecto final.

---

## 📁 Estructura del Proyecto

Adicional a los documentos de arriba, el proyecto está subdividido en dos directorios principales:

- **`demo/`** 🚀 
  Contiene la demostración funcional para el análisis técnico. Aquí encontrarás los microservicios Java/Spring Boot representando a **las 2 APIS: una diseñada bajo la arquitectura SOAP y otra bajo la arquitectura REST**, además de su propio `docker-compose.yml` para levantarlas.

- **`docs/`** 📚
  Aquí se alberga todo el historial o **proceso de documentación del proyecto**. Desde la base del refinamiento de las historias, los *prompts* utilizados en el uso de las Gemas, hasta la carpeta de sustentación `ppt` con las diapositivas (PDF) requeridas para la presentación del recurso.
